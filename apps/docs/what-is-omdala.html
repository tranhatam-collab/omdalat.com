What is OMDALA

OMDALA is the global system layer. It owns the shared docs surface, the runtime app surface, the trust framework, and the architecture that allows city nodes such as OMDALAT to exist without becoming fragmented products.

In the locked architecture, OMDALAT is the public Da Lat city node, while OMDALA owns the docs and app layers. This keeps the user journey clear: public discovery on OMDALAT, system understanding in OMDALA docs, and action in the OMDALA app runtime.

Global docs layer

Policies, operator guidance, trust logic, and system explanation belong under `docs.omdala.com`.

Global runtime layer

User actions, dashboard work, proofs, and system interaction belong under `app.omdala.com`.

City nodes

Each city node, such as OMDALAT, can keep a strong local identity while routing people into one shared system layer.

Why this matters

One global docs/app layer reduces brand split, user confusion, SEO dilution, and implementation drift.
