Overview

OMDALA Documentation

Global system docs, city-node context, and the path from public web into runtime action

OMDALAT uses a clean public path: omdalat.com -> docs.omdala.com -> app.omdala.com. The public site at omdalat.com introduces the Da Lat city node. The docs surface at docs.omdala.com explains the global logic, system rules, role pathways, and trust model before a user enters runtime action.

What this layer does

Make the system legible before someone enters it

  • Explain what OMDALA is and how OMDALAT fits into it.
  • Clarify the city-node concept before runtime entry.
  • Help each role understand docs, trust, and app behavior.
  • Strengthen trust through calm documentation rather than hype.

What this split does

Public web, docs, and app each have a different job

  • OMDALAT public web attracts and explains the city node.
  • OMDALA docs explain system meaning and policy.
  • OMDALA app handles runtime action.
  • Each layer stays clear instead of being mixed into one surface.
