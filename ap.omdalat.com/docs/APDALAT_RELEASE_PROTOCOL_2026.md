# APDALAT_RELEASE_PROTOCOL_2026
## Version: LOCKED
## Status: Production-ready
## Scope: ap.omdalat.com releases
## Required by: Founder / PM / Dev / Content / SEO / QA / Codex
## Purpose: Define the minimum release protocol for Ap Dalat
---
## 0. RELEASE RULE
Không claim release xong khi:
* chưa usable
* chưa verify live
* chưa có handoff
* chưa rõ blocker còn lại là gì

---
## 1. RELEASE LEVELS
### R1 Foundation
* Homepage
* Category hubs
* Core templates
* About
* Om Ap Dalat page
* Initial live deploy

### R2 Editorial Seed
* first story set
* people pages
* place pages
* image essays

### R3 Integrity
* deeper SEO
* search
* topic pages
* analytics refinement

---
## 2. PRE-RELEASE CHECK
Before release:
1. Docs locked
2. Content source updated
3. QA checklist run
4. URLs known
5. Handoff file written

---
## 3. HANDOFF FORMAT
Every release handoff must include:
1. Scope
2. Location
3. Implementation Summary
4. Completed
5. Not Included
6. Known Issues
7. Test Instructions
8. Next Steps

---
## 4. POST-RELEASE CHECK
After release:
1. Open live URL
2. Verify homepage
3. Verify one page per type
4. Verify images
5. Verify locale switch
6. Verify metadata on at least one VI and one EN page

---
## 5. DEFINITION OF DONE
A release is done only when live behavior matches the intended scope and the next person can understand the state without asking the Founder to reconstruct it.
