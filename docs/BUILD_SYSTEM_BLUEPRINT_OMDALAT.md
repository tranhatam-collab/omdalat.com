# BUILD SYSTEM BLUEPRINT
# OMDALAT.COM

Version: 1.0  
Status: Strategic build blueprint  
Scope: OMDALAT only  
Rule: This document defines the build direction for `omdalat.com` as an independent system. Any future connection to `omdala.com`, `*.nhachung.org`, or other external sources is only a future-ready option, not a present dependency.

---

# 1. SYSTEM DEFINITION

OMDALAT is a Local Opportunity and Income Operating Layer for Da Lat.

OMDALAT is not:
- a tourism blog
- a simple city guide
- a generic job board
- a random marketplace
- a social feed product

OMDALAT is:
- a city-node operating layer
- a local economic activation system
- a place where people, places, skills, programs, and trust become real income and real coordination
- a system that supports both online and offline value creation in Da Lat

The system must help members, hosts, experts, and local operators:
- see opportunity
- package value
- find the right match
- run programs
- create repeat revenue
- build trust over time

---

# 2. PRIMARY OUTCOME

The system must support these income outcomes:

- Member target: about `1,000 USD/month` stable
- Host target: about `2,000 to 5,000 USD/month`
- Expert target: about `2,000 to 5,000 USD/month`

OMDALAT should achieve this by activating:
- online work
- local services
- hosted programs
- workation and monthly stays
- expert-led education and advisory
- team-based delivery
- hybrid online/offline income

---

# 3. STRATEGIC POSITIONING

The long-term positioning of OMDALAT should be:

`The local operating layer that helps Da Lat turn people, places, skills, and community value into trusted income and long-term opportunity.`

Short positioning line:

`Turn Da Lat life into real value.`

Expanded positioning line:

`OMDALAT helps people, places, and communities in Da Lat turn local trust, real spaces, useful skills, and shared activity into income, programs, and repeat value.`

# 16. DEV PRIORITY ORDER

The correct priority order is:

1. identity + trust basics  
2. place and profile quality  
3. offer and request system  
4. bookings and program basics  
5. income graph and recurring offers  
6. admin moderation and operations  
7. payouts and partner flows  
8. future connectors

Do not reverse this order.

The most important KPIs are:

- number of active members with packaged offers
- number of verified hosts
- number of experts with bookable offers
- number of active programs
- number of completed bookings
- repeat booking rate
- proof completion rate
- trust growth rate
- average member income
- average host income
- average expert income

---

# 18. WHAT MUST NEVER HAPPEN

- Do not turn OMDALAT into a tourism-only site.
- Do not turn OMDALAT into an empty listing board.
- Do not depend on future integrations to make the current system work.
- Do not ship a marketplace without proof and trust.
- Do not let users wander without clear offer packaging.
- Do not let hosts depend only on one-off bookings.
- Do not let experts depend only on 1:1 consulting.

---

# 19. FINAL DECISION RULE

Every future feature should be tested against these questions:

- Does it help local people create real value?
- Does it improve income, trust, or coordination?
- Does it strengthen Da Lat as a living and working node?
- Does it help members, hosts, or experts move toward repeat revenue?
- Does it preserve OMDALAT as an independent system?

If not, it is not core.

---

# 20. FINAL SYSTEM DEFINITION

OMDALAT is a local operating system for turning Da Lat life into real opportunity, trusted programs, repeat income, and long-term economic value.

It should be built first as an independent city-node system.

It may connect to larger systems later.

But first, it must become useful, trusted, and economically real on its own.
