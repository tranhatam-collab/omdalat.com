0. Tuyên bố kích hoạt

Từ thời điểm này, Om Dalat chính thức áp dụng bộ chuẩn điều hành, build, song ngữ, SEO, frontend, QA/DevOps, và AI execution vào hệ tài liệu và cách làm việc của team.

Việc áp dụng này không thay thế brand lock của Om Dalat.
Việc áp dụng này không thay thế access model đã chốt.
Việc áp dụng này không thay thế ranh giới:

* `omdalat.com` = public web và lối vào hệ
* `app.omdalat.com` = app/member runtime
* `ap.omdalat.com` = lớp editorial độc lập

Nó bổ sung một lớp điều hành bắt buộc để mọi team cùng làm đúng một chuẩn production.

4. Những thay đổi bắt buộc từ hôm nay

4.1 Về product và docs

Mọi file spec mới trong `docs/` phải ghi rõ:

* owner
* scope
* status
* version
* route hoặc surface liên quan
* acceptance criteria

4.2 Về public language

Tất cả public text của Om Dalat phải đi theo nguyên tắc:

* tiếng Việt có dấu đầy đủ
* tiếng Anh tự nhiên, không dịch máy
* không hype
* không lộ nhãn kỹ thuật nội bộ
* không để một page live khi VI/EN lệch nghĩa

4.3 Về SEO

Mỗi page public phải có:

* H1 duy nhất
* title riêng theo ngôn ngữ
* meta description riêng theo ngôn ngữ
* canonical đúng
* hreflang đúng
* alt text đúng ngôn ngữ
* internal links đúng locale

4.4 Về frontend

Mọi thay đổi UI phải bám:

* token màu
* typography scale
* spacing scale
* radius scale
* image rules
* motion rules
* accessibility rules
* mobile behavior

4.5 Về QA / release

Mỗi release hoặc pre-live gate phải có:

* commands used
* pages tested
* affected files
* result
* risks
* rollback note
* approval recommendation

4.6 Về AI/dev execution

Mỗi task cần:

* phân loại độ lớn
* phân loại mức rủi ro
* chọn lane model phù hợp
* evidence trước khi claim done
* không trộn nhiều task không liên quan trong một session
