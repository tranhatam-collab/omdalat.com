# OMDALAT FOUNDER DECISION BRIEF
# OMDALAT.COM

Version: 1.0  
Purpose: One-page founder strategy brief to decide what OMDALAT keeps now, delays, rejects, or treats only as a future connector.

---

# 1. CORE DECISION

OMDALAT remains a standalone Da Lat city-layer system focused on:
- local opportunity
- income creation
- trusted places
- host and expert activation
- online and offline value creation

OMDALAT should not depend on any outside platform to become useful.

---

# 2. KEEP NOW

These directions are core and should stay inside the current OMDALAT build.

## Keep

- `People layer`
  members, hosts, experts, operators, connectors

- `Place layer`
  homestays, coliving, coworking, cafes, studios, retreat spaces

- `Income layer`
  offers, requests, recurring packages, earnings paths

- `Program layer`
  workshops, cohorts, retreats, residencies, local events

- `Trust layer`
  proof, completion, endorsements, verification, reliability

- `Da Lat-specific positioning`
  living, working, learning, hosting, creating, and earning in Da Lat

- `Bilingual growth direction`
  public communication designed for both Vietnamese and future English expansion

---

# 3. HOLD FOR LATER

These directions are useful but should not be pushed into current execution unless directly needed.

## Hold

- advanced payout orchestration
- complex recurring billing systems
- full partner network automation
- advanced operator dashboards beyond MVP needs
- deep analytics beyond basic earning and activation metrics
- broad city expansion beyond Da Lat

---

# 4. DO NOT BUILD NOW

These directions are not core for the current OMDAT phase and should not take focus away from real value loops.

## Do not build now

- empty job marketplace
- social feed-first product
- complex multi-vendor marketplace settlement
- global location network inside OMDALAT
- dependency on outside data sources for core functionality
- speculative integration work without active approval

---

# 5. FUTURE CONNECTOR ONLY

These may become powerful later, but they are not active dependencies now.

## Future connector only

- `omdala.com`
  future global demand, category, partner, and scale layer

- `lamviecmuonnoi.com`
  future remote work, earning pathways, location intelligence, and credibility amplification source

- `*.nhachung.org`
  future trust, place, community, or program source if and when approved

- local schools, coworking networks, host networks, creator networks, retreat operators

Rule:
- no present hard coupling
- no current roadmap dependency
- no launch blocker

---

# 6. WHAT OMDALAT MUST PROVE FIRST

Before any real connector work, OMDALAT should prove:

- members can package real offers
- hosts can generate structured revenue
- experts can run paid programs
- trust improves discoverability and conversion
- programs and bookings create repeat value
- Da Lat density creates better outcomes than generic platforms

---

# 7. FOUNDER DECISION CHECK

- [ ] Keep OMDALAT fully standalone for current build
- [ ] Keep income + trust + program direction as the core
- [ ] Hold external integration until explicitly approved
- [ ] Use future connectors only as strategic orientation, not delivery dependency

---

# 8. FINAL DECISION RULE

If a future feature helps OMDALAT create real local income, trusted activity, and repeat value in Da Lat, keep it.

If it only sounds big, broad, or globally impressive but weakens focus, delay it.

If it turns OMDALAT into a dependent shell around another system, reject it.
