# LILY WORKSPACE AND REMOTE WORK OS 2026

## 1. Purpose

The workspace exists to support focused work, learning and verified project delivery. It is not a co-working marketing claim until internet, desk capacity, acoustics, power and safety are evidenced.

## 2. Workspace zones

- quiet focus zone;
- shared collaboration table;
- small meeting/video-call area;
- café corner;
- equipment/storage area;
- garden break area.

Each zone has capacity, hours, noise rule, bookability, accessibility, equipment and current status.

## 3. Operating rules

- Quiet hours are defined in house rules and calendar.
- Calls use designated zones or headphones.
- Shared meetings require calendar booking.
- Private/confidential client calls cannot be recorded without consent.
- Residents do not access another resident’s screen, files or devices.
- Project data stays in approved systems.
- Personal and project accounts must use MFA where supported.
- Shared computers are not used for passport, banking or restricted project data unless managed securely.

## 4. Work modes

### Personal remote work
Resident works for their own overseas or independent client. Lily provides space only and does not become employer, sponsor or party to the work.

### Learning assignment
Output is educational. No employment promise. Compensation status is `learning`.

### Volunteer contribution
Permitted only when lawful, genuinely voluntary, written and not used to replace paid labour improperly.

### Paid project task
Requires project owner, written scope, compensation, acceptance rubric, payment route, IP/confidentiality terms and work-authorisation gate where applicable.

## 5. Proof of work

Each task uses:

```text
Brief
→ Acceptance
→ Work log
→ Draft submission
→ Review
→ Revision
→ Accepted / Rejected with reason
→ Payment eligibility
→ Portfolio permission decision
```

Proof may include files, URLs, commit SHA, CMS entry, research sources, checklist, screenshot or reviewer sign-off. Time online alone is not proof of delivery.

## 6. Project assignment rules

- No hidden assignment through chat only.
- No task without responsible owner.
- No “urgent” label without deadline and priority reason.
- No access to production secrets by default.
- Least-privilege access expires after task closure.
- Residents can decline optional tasks without losing accommodation rights.
- Accommodation duties are not mixed with digital project compensation.

## 7. Portfolio system

A resident may publish accepted work only when:

- project owner approves disclosure;
- client/confidential data is removed;
- IP terms allow it;
- contribution is accurately described;
- no claim of employment or partnership is implied without evidence.

## 8. Workspace review metrics

- workspace availability;
- internet/power incident rate;
- quiet-hour compliance;
- meeting conflicts;
- accepted deliverables;
- revision cycles;
- access-control incidents;
- resident satisfaction;
- task payment completion.
