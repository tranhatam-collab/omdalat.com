# USER_ROLES_OMDALAT
## Version 1.0

---

# 1. Public Roles

## 1.1 Guest

Can:

- browse public pages
- view indexable entities
- read trust summaries

Cannot:

- create requests
- upload proofs
- access the local app

## 1.2 Member

Can:

- access local app basics
- manage own node
- create requests
- respond to participation flows

## 1.3 Verified Member

Can:

- access higher-trust participation flows
- appear with stronger trust markers

---

# 2. Entity Roles

## 2.1 Host

A member or team operating a place.

## 2.2 Expert

A member offering local expertise or service capacity.

## 2.3 Community Operator

A member managing a local community or recurring activity layer.

## 2.4 Place Operator

A member or team managing a public place record.

---

# 3. Admin Roles

## 3.1 Moderator

Can:

- review public content
- flag or limit visibility
- review proof submissions

## 3.2 Verifier

Can:

- review verification requests
- update verification status

## 3.3 Admin

Can:

- inspect all entities
- manage visibility states
- handle support and trust review

---

# 4. Permission Groups

- view
- create
- edit
- publish
- review
- verify
- moderate
- govern

---

# 5. Rules

- backend permission checks are mandatory
- no role change without audit trail
- verification actions must be reviewable
- moderation actions must carry timestamps and actor identity
