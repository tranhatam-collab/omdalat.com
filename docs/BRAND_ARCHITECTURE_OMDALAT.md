# OMDALAT Brand Architecture

Version: 1.0
Status: approved city brand direction

## 1. Brand role

OMDALAT is the first city implementation of OMDALA.

It is not a separate ideology and not an unrelated local brand.

It is:

- the first city node
- the first living lab
- the first density engine
- the first place where proof is visible

## 2. Positioning

Primary:

- OMDALAT
- The First Living Intelligence City

Secondary:

- Where OMDALA becomes real

## 3. Brand definition

OMDALAT should represent:

- real hosts
- real places
- real experts
- real communities
- real requests
- real proofs

## 4. What OMDALAT must never become

Do not turn it into:

- a travel site
- a tourism blog
- a local directory with no activity
- a decorative community page

## 5. Correct product lens

OMDALAT is:

- a city operating layer
- a local activation system
- a density and proof engine

## 6. Key surfaces

The first city layer should highlight:

- active places
- verified hosts
- ongoing requests
- upcoming activities
- trusted local nodes
- visible proofs

## 7. Relationship to OMDALA

Simple rule:

- OMDALA defines the system
- OMDALAT proves the system

## 8. Domain strategy

- `omdalat.com` = city homepage
- `app.omdalat.com` = local city experience

## 9. Design tone

OMDALAT should feel:

- alive
- credible
- local
- premium
- operational

It should feel warmer and more human than OMDALA, but still inherit the same calm and trusted system language.

## 10. Final brand rule

Every OMDALAT page should answer:

- what is active now
- who is real here
- what can happen in the city layer
